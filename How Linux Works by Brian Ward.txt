    1.3. The Kernel
    The kernel is in charge of managing tasks for four general system areas:
    - Processes: kernel is responsible for determing which processes can use CPU.
    - Memory: kernal needs to keep track of all memory.
    - Device drives: kernel is an interface between processes and hardware.
    - System calls and support: processes usually use system calls to communicate with the kernel.

    1.3.1. Process Managment
    Process managment describes starting, pausing, resuming, scheduling and terminating of processes.
    Concurrensy vs Parallelism
    Context switch - act of one process giving up control of the CPU to another.
    Time slice - minimal time, needed for a process to complete minimal significant computation between context switches.
    Kernel runs between process time slices during a context switch.

    1.3.2. Memory Managment
    Memory managment unit (MMU) - enables 'virtual memory'.

    1.3.3. Device Drivers and Managment
    Device drivers are usually acessible only to kernel.

    1.3.4. System Calls and Support
    System calls or syscalls perform tasks, that user processes are not able to do well or at all.
    fork() - when called by a process, the kernel creates a nearly identical copy of the process.
    exec() - the kernel loads and starts a program, replacing the current process.
    The kernel also supports user processes with features other than system calls, such as pseudodevices.


    1.4. User Space
    User space - main memory that the kernel allocate for the user processes.
    Despite user processes are the same from the kernel's point of view, we can divide them into different levels.


    1.5. Users
    Linux kernel supports traditional consept of a Unix user.
    User - an entity that can run processes and own files.
    Groups are sets of users.
    Primary goal of groups is to share file access to other members of a group.
